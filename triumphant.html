        Looks for data-tulip-clipboard inside the HTML clipboard fragment, base64-decodes it, and renders triggers as If/Then blocks.
        Action and condition types are mapped from a small known table (store_in_var, array_get_at_index, is_false, etc.) —
        unrecognized types fall back to a generic row so nothing is silently dropped. Expression strings (exprStr) are parsed from their internal AST;
        infix operators are best-effort, since only + has been confirmed against a real screenshot so far.
